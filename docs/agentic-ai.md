## Three Pillars

### 1. Custom Agent Development

We design and deploy AI agents tailored to your platform. Whether you are a sportsbook, a data feed provider, or a trading desk, we build agents that integrate with your existing systems and operate autonomously within the boundaries you define.

- **Six agent types** available: Odds Monitoring, Data Verification, Risk Assessment, Market Analysis, Trading Assistant, and Compliance
- **Proof of Concept delivered in 4 weeks** — a working agent connected to your environment, demonstrating core functionality
- **Production deployment in 8 weeks** — fully tested, monitored, and ready for live operation
- Agents are built on OddsFlow infrastructure but operate within your environment, under your control

Learn more: [Custom Agent Development](./custom-agents.md)

### 2. Agent-to-Agent (A2A) Protocol

Our open protocol defines how agents communicate across organizational boundaries. A sportsbook's risk agent can query a feed provider's data agent. A user's personal agent can request verified odds from multiple sportsbook agents simultaneously. The protocol ensures structured, verifiable, auditable exchanges.

- **Open protocol** — published specification, reference implementations, community governance
- **Structured data exchange** with standardized message formats for odds, markets, signals, and settlements
- **Mutual verification** — agents independently verify each other's claims against fair-value benchmarks
- **Reputation-based trust** — every interaction is scored, and agents with higher reputation are prioritized in the network

Learn more: [Agent-to-Agent Protocol](./agent-protocol.md)

### 3. Agent Reputation Network

Every agent on the OddsFlow network accumulates a cryptographic track record. Accuracy, latency, uptime, and consistency are continuously measured and scored. This reputation is portable and verifiable — third-party agents can query it before deciding whether to trust a counterpart.

- **Cryptographic track records** — immutable logs of every claim, verification, and interaction
- **Dynamic scoring from 0 to 100** — updated in real time based on agent performance across accuracy, latency, uptime, and consistency
- **Open protocol for third-party agents** — any agent, whether built by OddsFlow or independently, can participate in the Reputation Network and earn trust through performance
- Higher-reputation agents receive priority routing, better data access, and preferential treatment in trading negotiations
